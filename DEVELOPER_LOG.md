# 開發者日誌 - JoJoTrading 股票分析工具

## 2025-05-10

**本日重點：** 整合資產負債表數據以優化DCF估值模型中的FCFE計算，並修正財報單位錯誤。

**詳細記錄：**

1.  **目標設定：**
    *   根據使用者提供的 Excel 估值模型，其核心是基於預期未來「獲利」進行折現。
    *   我們的目標是讓 Python DCF 模型更接近標準 FCFE 計算，第一步是嘗試整合資產負債表數據以計算「營運資本變動 (ΔWC)」。

2.  **資產負債表數據整合嘗試：**
    *   在 `data_handler.py` 中：
        *   新增了 `BALANCE_SHEET_API_PATH_PREFIX` 和 `BALANCE_SHEET_FIELD_NAMES_MAP` (初步定義)。
        *   新增了 `get_balance_sheet_data_for_stock` 函式，用於從 TWSE OpenAPI (`t187ap07_L_xx`) 下載資產負債表數據。
        *   修改了 `fetch_balance_sheet_items_from_downloaded` 函式，使其嘗試提取最新兩期 (T0 和 T-1) 的應收帳款、存貨和應付帳款。
    *   在 `jojo_state_machine.py` 的 `DataFetchState` 中：
        *   加入了調用上述新函式的邏輯，並將提取的兩期資產負債表項目存入 `financials` 字典。
    *   在 `data_handler.py` 的 `calculate_dcf_valuation` 中：
        *   加入了根據 T0 和 T-1 期資產負債表項目計算 WC_t0, WC_t1, ΔWC 及 ΔWC_ps 的邏輯。
        *   初步將 FCFEps 調整為 `(淨利歸母 / 流通股數) - ΔWC_ps`。

3.  **遭遇數據源限制與策略調整：**
    *   **問題發現**：使用者協助確認，TWSE OpenAPI 的資產負債表端點 (`t187ap07_L_ci`) 提供的是**高度匯總的數據**，不包含計算 ΔWC 所需的「應收票據及帳款淨額」、「存貨」、「應付票據及帳款」等明細科目。
    *   **臨時策略調整**：
        *   由於無法從當前 API 精確計算 ΔWC，決定將 FCFEps 的代理計算**明確退回到直接使用「每股淨利」** (`淨利歸母 / 流通股數`)。這與使用者原始 Excel 模型的思路更為接近。
        *   相應地，在 `calculate_dcf_valuation` 中移除了 ΔWC 的計算邏輯。
        *   在 `DataFetchState` 中暫時註釋掉了獲取和處理資產負債表數據的程式碼，以避免不必要的 API 請求和處理目前無法有效利用的數據。

4.  **修正關鍵的財報金額單位錯誤：**
    *   **問題發現**：通過測試日誌觀察到，API 返回的「淨利（淨損）歸屬於母公司業主」和「營業收入」等金額是以「千元」為單位，而「流通股數」是「股」，導致計算出的每股數值（如 FCFEps）被嚴重低估1000倍，進而使估值結果（內在價值、潛在報酬率）嚴重偏低。
    *   **解決方案**：在 `data_handler.py` 的 `fetch_stock_financials_from_downloaded` 函式中，將提取到的「淨利歸母」和「營收」數值**乘以 1000**，將其單位從「千元」轉換為「元」。

5.  **測試與驗證：**
    *   在修正金額單位後，再次執行應用程式進行測試（以「電子零組件業」為例）。
    *   **結果**：
        *   提取的「淨利歸母(元)」和「營收(元)」數值正確放大1000倍。
        *   計算出的「FCF_EPS (基於淨利/股數)」回到了合理的數量級。
        *   最終的「內在價值」和「潛在報酬率」也因此變得更為合理。
        *   程式能夠篩選出符合潛在報酬率條件的股票 (例如，在測試中選出了19支電子零組件業股票)。

**當前狀態：**

*   應用程式目前基於「淨利折現」的簡化DCF模型運行。
*   由於財報金額單位問題已修正，估值結果的數量級相對合理，能夠提供初步的參考。
*   獲取和處理資產負債表明細數據的邏輯已暫時移除/註釋，等待未來找到更合適的詳細數據源。

**待辦事項/未來方向：**

*   使用者將嘗試尋找能提供更詳細財務報表（包含資產負債表、現金流量表明細科目）的數據源。
*   若找到合適數據源，將重新啟用並完善 FCFE 的計算邏輯，納入精確的 ΔWC、資本支出、折舊等。
*   考慮引入股利折現模型 (DDM) 作為另一種估值方法（需股利數據）。
*   持續優化數據獲取、錯誤處理和使用者介面。
