# 常見問題與解答 (FAQ)

## 專案基礎問題 🏗️

### Q1: JoJo Trading System 是什麼？
**A**: JoJo Trading System 是一個基於 Python + Streamlit 開發的台股 DCF (現金流折現) 估值分析系統。主要功能包括：
- 自動抓取台股財務數據
- 多情境 DCF 估值計算
- 互動式 Web 介面
- 投資決策支援工具

### Q2: 為什麼叫 "JoJo Trading System"？
**A**: "JoJo" 是專案的代號名稱，代表輕鬆且友善的投資分析工具，讓投資分析不再是專業人士的專屬，而是人人都能使用的智能工具。

### Q3: 這個系統適合什麼人使用？
**A**: 
- **個人投資者**: 進行股票投資前的基本面分析
- **金融從業人員**: 快速進行估值分析與比較
- **學術研究者**: 研究台股估值模型與方法
- **開發者**: 學習金融數據分析與 Web 開發

### Q4: 系統的主要價值是什麼？
**A**:
- **時間節省**: 自動化數據收集與計算，節省 90% 手動作業時間
- **準確性提升**: 標準化計算流程，減少人為錯誤
- **一致性保證**: 統一的估值標準與參數
- **決策支援**: 提供多情境分析幫助投資決策

## 技術相關問題 🔧

### Q5: 系統使用什麼技術開發？
**A**: 
- **後端**: Python 3.8+, pandas, numpy
- **前端**: Streamlit (Web 框架)
- **數據源**: FinMind API, TWSE 官網
- **圖表**: Plotly 互動式圖表
- **快取**: 本地檔案快取系統

### Q6: 為什麼選擇 Streamlit 而不是 Flask 或 Django？
**A**: 
- **快速開發**: Streamlit 專為數據應用設計，開發速度快
- **內建組件**: 豐富的數據視覺化與互動組件
- **學習曲線**: 對非 Web 開發背景的數據科學家友善
- **維護成本**: 代碼簡潔，維護成本低

### Q7: 系統的效能如何？
**A**:
- **DCF 計算**: 單一公司 < 5 秒
- **頁面載入**: < 2 秒 (有快取時)
- **API 調用**: 平均 1.5 秒
- **記憶體使用**: 基礎 180MB，峰值 520MB

### Q8: 如何處理大量數據？
**A**:
- **本地快取**: 減少重複 API 調用
- **批量處理**: 支援多公司同時分析
- **分頁載入**: 大數據集分批載入
- **壓縮儲存**: 快取數據壓縮存放

## 功能使用問題 📊

### Q9: DCF 計算的準確性如何？
**A**: 
- **理論基礎**: 採用標準 DCF 估值理論
- **參數校準**: 基於台股歷史數據校準參數
- **驗證測試**: 手動驗證 20+ 知名公司，誤差 < 5%
- **持續改善**: 根據市場變化持續調整模型

### Q10: 支援哪些股票？
**A**:
- **上市公司**: 台股所有上市公司
- **資料完整性**: 優先支援財報完整的公司
- **特殊情況**: 金融股、REIT 需特別處理
- **更新頻率**: 公司清單每週自動更新

### Q11: 如何處理缺失或異常數據？
**A**:
- **缺失數據**: 使用同期其他公司平均值或前期數據
- **異常值檢測**: 自動識別並標記異常數值
- **人工確認**: 重要異常會提示人工檢查
- **數據來源**: 多數據源交叉驗證

### Q12: 情境分析如何設定？
**A**:
- **樂觀情境**: 關鍵參數上調 20%
- **保守情境**: 關鍵參數下調 20%
- **中性情境**: 使用標準參數
- **自定義**: 支援手動調整關鍵參數

## 安裝與部署問題 🚀

### Q13: 系統需求是什麼？
**A**:
- **作業系統**: Windows 10+, macOS 10.15+, Linux Ubuntu 18.04+
- **Python**: 3.8 或更新版本
- **記憶體**: 最少 4GB (建議 8GB+)
- **硬碟**: 2GB 可用空間
- **網路**: 穩定網際網路連線

### Q14: 如何快速啟動系統？
**A**:
```bash
# 1. 克隆代碼
git clone <repository>
cd jojo_trading

# 2. 安裝依賴
pip install -r requirements.txt

# 3. 啟動應用
streamlit run main_app.py

# 4. 開啟瀏覽器
http://localhost:8501
```

## 數據與 API 問題 🌐

### Q21: FinMind API 有什麼限制？
**A**:
- **免費額度**: 每日 1000 次 API 調用
- **註冊優惠**: 註冊後可獲得更多額度
- **付費方案**: 專業用戶可購買更高額度
- **備援方案**: 系統支援多數據源切換

### Q22: 如何申請 FinMind API 金鑰？
**A**:
1. 前往 FinMind 官網註冊帳號
2. 在控制台取得 API Token
3. 設定到 `config/local_config.json`
4. 重新啟動應用即可使用

### Q23: API 調用失敗怎麼辦？
**A**:
- **檢查網路**: 確認網際網路連線正常
- **檢查額度**: 確認 API 調用額度未用盡
- **重試機制**: 系統會自動重試失敗的請求
- **快取備援**: 使用本地快取數據作為備援

### Q24: 數據更新頻率如何？
**A**:
- **股價數據**: 每 15 分鐘更新 (交易時間)
- **財務數據**: 財報發布後 2-3 天
- **公司基本資料**: 每週自動更新
- **快取策略**: 根據數據類型設定不同過期時間

## 投資決策問題 💼

### Q25: DCF 結果可以直接作為投資決策嗎？
**A**: 
**不建議直接依賴單一指標做投資決策**。DCF 分析應該結合：
- 技術分析指標
- 市場情緒與總體經濟
- 公司治理與競爭優勢
- 風險評估與個人風險承受度

### Q26: 系統計算結果的免責聲明？
**A**: 
- **僅供參考**: 計算結果僅供投資參考，不構成投資建議
- **風險自負**: 投資決策與風險需自行承擔
- **數據準確性**: 雖力求準確但不保證數據 100% 正確
- **市場風險**: 股市投資存在固有風險，過去績效不代表未來表現

### Q27: 如何解讀 DCF 分析結果？
**A**:
- **內在價值 vs 市價**: 比較計算出的內在價值與目前市價
- **安全邊際**: 建議買入價格應有 20-30% 安全邊際
- **情境分析**: 參考樂觀/保守情境的價值區間
- **敏感性分析**: 了解關鍵參數變化對結果的影響

### Q28: 適合什麼投資風格？
**A**:
- **價值投資**: 最適合尋找被低估股票的價值投資者
- **長期投資**: DCF 分析適合長期持有策略
- **基本面分析**: 重視公司基本面的投資者
- **不適合**: 短線交易、技術分析導向的投資風格
