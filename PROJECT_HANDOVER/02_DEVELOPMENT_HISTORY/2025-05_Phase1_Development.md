- **資料來源**: FinMind API
- **程式語言**: Python 3.8+
- **資料處理**: Pandas, NumPy
- **架構模式**: 模組化設計

## 📅 2025年5月16日 - XBRL技術研究

### 🔬 XBRL財報解析技術預研

**研究目標**: 評估直接從XBRL解析財報資料的可行性

**技術評估**:
- **優勢**: 資料來源權威、標準化程度高
- **挑戰**: 解析複雜度高、格式不一致
- **風險**: 開發週期長、維護成本高

**結論**: 暫時採用 FinMind API，保留 XBRL 作為未來升級選項

## 📅 2025年5月17日 - 成長股判定機制設計

### 🎯 成長股判定機制需求細節補充與決策紀錄

**核心需求**:
- 自動化成長股判定機制
- 可調整的智能閾值
- 多維度指標評估

**設計決策**:
- **預設智能閾值**: 近3年營收CAGR > 15%、EPS CAGR > 15%、ROE > 15%
- **UI設計**: 多項成長指標勾選（checkbox）
- **邏輯組合**: AND/OR 邏輯組合
- **異常處理**: 數據不足時標記「N/A」並說明

**實作規格**:
```python
# 成長股判定標準
GROWTH_CRITERIA = {
    'revenue_cagr': 0.15,      # 營收複合成長率 > 15%
    'eps_cagr': 0.15,          # EPS複合成長率 > 15%
    'roe': 0.15,               # ROE > 15%
    'profit_margin_trend': 'positive'  # 利潤率趨勢向上
}
```
